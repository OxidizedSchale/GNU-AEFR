## 🧭 ナビゲーション: GNA:AEFRの哲学

> **GNA:AEFR** は、大衆を喜ばせるために設計された凡庸なアプリケーションではありません。現在、コアカーネルの高速反復フェーズにあり、純粋なRustロジック上に構築されたハイパフォーマンスなエディタインスタンスです。**計算の世界において、最短のパスこそが最強です。**
> 
> 普遍的な互換性や「箱出しの状態」での低い参入障壁を求めるのであれば、**AA** を使用してください。もし、極限の実行効率、絶対的な創作の自由、そしてハードコアなコミュニティ主導のメンテナンスを追求するなら、**GNA:AEFR** の世界へようこそ。

*   **クロスプラットフォームの主権**: GNU/Linux、Android、macOS、Windowsをネイティブサポートし、異なるOS間でも一貫した優れたエクスペリエンスを保証します。

## 🚀 はじめに

> *「リリース？ 本物のハッカーはソースからコンパイルするものだ」 ;-)*

[**Releases**](https://github.com/OxidizedSchale/GNA-AEFR/releases) ページから最新のソースコードを取得するか、特定のプラットフォーム向けにプリコンパイルされたバイナリをダウンロードしてください。

**GNA:AEFR** は、**グラフィカルユーザーインターフェース (GUI)** と **コマンド駆動** ロジックを深く統合したインタラクションモデルを採用しています。

*   **デスクトップ**: フル機能のグラフィカルコントロールパネルの使用を推奨します。
*   **モバイル**: 外部リソースのインポートには、現在コンソールコマンドによる精密なスケジューリングが必要です。

インターフェース左上の `[SHELL]` ボタンをクリックして、内蔵のハードコア・デバッグコンソールを呼び出してください。

<details>
<summary><strong>📖 クリックして展開: GNA:AEFR コマンドリファレンスマニュアル</strong></summary>

### 1. ビジュアル & シーン

*   **背景の読み込み**
    *   **コマンド**: `BG <image_path>`
    *   **説明**: 背景を即座に切り替えます。`.jpg`, `.png`, `.webp` をサポート。
    *   **例**: `BG C:\Assets\BlueArchive\BG_Classroom.png`

*   **キャラクターの読み込み**
    *   **コマンド**: `LOAD <slot_ID> <.atlas_path>`
    *   **説明**: スロット `0` から `4` にSpineリソースを読み込みます。成功すると、SHELLはそのキャラクターで使用可能なアニメーションの全リストを返します。
    *   **例**: `LOAD 0 D:\Assets\Shiroko\Shiroko_Home.atlas`

### 2. モーション & パフォーマンス

*   **アニメーションの切り替え**
    *   **コマンド**: `ANIM <slot_ID> <animation_name> [loop: true/false]`
    *   **説明**: 指定したスロットのキャラクターのアニメーションを切り替えます。`true` で強制ループ、`false` で単発再生。アニメーション名は読み込み時に返されたリストと完全に一致させる必要があります。
    *   **例**:
        ```bash
        ANIM 0 Start_Idle_01 true    # シロコがアイドル状態をループ
        ANIM 1 Attack_Normal false   # スロット1のキャラクターが通常攻撃を1回実行
        ```

### 3. ストーリーテリング & ダイアログ

*   **ダイアログの送信**
    *   **コマンド**: `TALK <名前>|<所属>|<内容>`
    *   **説明**: タイプライターエフェクトを統合した標準的なダイアログボックスをレンダリングします。**パラメータはパイプ記号 `|` で区切る必要があります。**
    *   **例**:
        ```bash
        TALK シロコ|アビドス|先生、銀行を襲撃しに行かない？
        TALK アロナ|シャーレ|先生、お仕事中にサボらないでください！
        ```

### 4. オーディオシステム

*   **BGMの再生**
    *   **コマンド**: `BGM <audio_path>`
    *   **説明**: オーディオエンジンに指令を出し、オーディオファイルを非同期で読み込み、ループ再生します。
    *   **例**: `BGM D:\Music\Unwelcome_School.mp3`

*   **再生停止**
    *   **コマンド**: `STOP`
    *   **説明**: 現在アクティブなオーディオ出力ストリームを強制終了します。

</details>

### 💡 プロのヒント

*   **パスの事前処理**: Windowsユーザーはファイルパスを直接貼り付けることができます。GNAが引用符やエスケープ文字を自動的に処理します。Android/Termuxユーザーは、`/sdcard/` から始まる絶対パスを使用していることを確認してください。
*   **スケジューラの保証**: 独自の「ジェントルマンズ・スケジューラ (Gentleman's Scheduler)」により、UIスレッドは物理的に隔離されています。画面上に5人のキャラクターがいて計算負荷が最大になっても、インターフェースは完全にバターのように滑らかなままです。
*   **リアルタイム監査**: リソースの読み込み状態やアニメーションのパースログをリアルタイムで監査するために、SHELLウィンドウを表示したままにすることをお勧めします。

## 🤝 貢献ガイド

<details>
<summary><strong>クリックして展開: GNA常任委員会 貢献者規約</strong></summary>

### 技術スタックの純粋性
本プロジェクトでは、アーキテクチャの純粋性を損なうあらゆる行為を厳格に禁止します。コアビジネスロジックは100% **Rust** で実装されなければなりません。

*   **原則的拒絶**: C++ランタイムや重量級フレームワーク（Qt、Unityなど）との複雑なJNI/FFI相互作用を導入する提案。
*   **例外的許可**: 低レイヤのシステムCライブラリ（グラフィックスAPI、オーディオバックエンド）に対するRustのセーフティラッパーのみ許可されます。その場合：
    *   成熟したコミュニティの `-sys` バインディングを優先すること。
    *   `unsafe` コールを書く必要がある場合は、以下の安全証明基準を厳格に遵守すること。
    *   最終的な目標は、非安全なコールを完全にデカップリングし、透過的で安全なRust APIを提供することである。

### Unsafeガイドライン: パフォーマンスという名の諸刃の剣の制約
> `unsafe` は、Rustが開発者に与えたパフォーマンスの刃です。私たちの原則は「必要でない限り使わない。使う場合は、万全を期す」です。

**核心原則: 必要性の証明**
`unsafe` ブロックを含むPRは、コード上のコメントで必要性の証明を提供しなければなりません：
1.  **理由**: なぜ安全なRustでは現在の低レイヤの要件を満たせないのかを明確にすること。
2.  **不可代替性**: 既存の安全なライブラリでは同じ機能を実現できないことを示すこと。
3.  **安全境界**: 開発者がコンパイラに対して約束しているメモリ不変条件を明確に定義すること。

</details>

## 🏛️ アーキテクチャの哲学

<details>
<summary><strong>クリックして展開: GNA:AEFRを駆動するハードコアな設計の深掘り</strong></summary>

### ジェントルマンズ・スケジューラ: 計算レベルにおける階級分離
> v0.8以降、我々はOSのデフォルトのスケジューリングロジックを信頼していません。それは、凡庸な「公平性」のためにリアルタイムレンダリングの決定論を犠牲にすることが多いからです。

*   **N-2 戦略**: GNAは物理コア数 `N` を強制的に検出し、`N-2` 個のコアを専用の計算ゾーンとして隔離します。
    *   **レンダーコア**: メインスレッドのGUI描画専用に1コアを排他的にロックします。
    *   **オーディオコア**: CPUスパイクによる音飛びを完全に排除するため、システムオーディオの「バッファスペース」として1コアをロックします。
    *   **計算ゾーン**: 残りのすべてのコアはWorkerに引き渡され、Rayonスレッドプールを介してSpineのスケルトンスキニングから物理的なパフォーマンスを最後の一滴まで絞り出します。

*   **同期ブロッキングモデル**: ウェブ開発に見られる安価で制御不能な「非同期」を拒絶します。
    *   **空間的整合性**: メインスレッドはUpdateフェーズ中に結果を同期的に待機し、すべてのフレームの視覚要素が時空を超えて絶対的に整列していることを保証します。
    *   **ワークステアリング**: 低コア数のデバイスでは、メインスレッドが積極的に計算タスクを引き継ぎ、電力利用率が物理的限界に達するようにします。

### GUI設計原則
*   すべてのコンポーネントは、`egui` の **イミディエイト・モード (Immediate Mode)** 哲学に厳格に従わなければなりません。

</details>
